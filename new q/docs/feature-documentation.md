# توثيق الـ Features

هذا الملف يوثق الـ features الأساسية في مشروع Quranic Pomodoro Next بشكل عملي ومباشر.

## 1) نظرة سريعة على المنتج

التطبيق يجمع بين:

- جلسات تركيز (Pomodoro) لدراسة القرآن.
- تجربة قراءة مرنة (أرباع أو صفحات مصحف).
- تلاوة صوتية مع تتبع الكلمة النشطة أثناء التشغيل.
- صفحة تفسير مرتبطة بالآية مع تمييز بصري للمحتوى.
- إعدادات محفوظة وإحصائيات استخدام.

## 2) Feature: Focus Workspace

الهدف:

- إدارة جلسات الدراسة والاستراحة.
- ضبط مدة الدراسة/الاستراحة والهدف اليومي.
- الانتقال السلس بين وضع الدراسة والقراءة.

السلوك:

- المستخدم يختار مدة الدراسة والراحة من واجهة سريعة.
- المؤقت يعتمد على `timer-store` ويطبق التغييرات بشكل متزامن.
- يتم حفظ الإعدادات عبر `settings-store` لتبقى بعد إعادة فتح التطبيق.

نقاط تقنية:

- الشاشة مبنية على مكونات تفاعلية مع dropdowns صغيرة.
- يتم تهيئة صوت التنبيه عالميًا (alarm bridge) لدعم transitions.

## 3) Feature: Reader Rub Mode

الهدف:

- قراءة مجموعة أرباع متتالية مع تنقل سريع.
- دعم تلاوة الربعة كاملة + تظليل الكلمة حسب التوقيت.

السلوك:

- تحميل المحتوى من `/api/rub`.
- عند تشغيل التلاوة: يتم جلب الصوت + word timings، ثم تشغيل gapless sequencer.
- عند الانتقال للربع التالي/السابق: يتم إيقاف التلاوة الحالية وتنظيف الحالة.

نقاط تقنية:

- إدارة الحالة من `reader-store` و`settings-store`.
- تجميع الصفحات داخل الربعة وعرضها عبر `RubPageLayout`.

## 4) Feature: Reader Mushaf Page Mode

الهدف:

- قراءة صفحات مصحف QCF مع تحكم في التكبير والتنقل.
- تشغيل تلاوة الصفحة مع تتبع الآية والكلمة.

السلوك:

- تحميل الصفحة/الصفحات من `/api/page`.
- تشغيل تلاوة الصفحة من `/api/page_recitation`.
- تحميل توقيت الكلمات من `/api/page_word_timings`.

نقاط تقنية:

- يدعم عرض 1-3 صفحات حسب الإعداد.
- يدعم overlay controls للتكبير/التصغير والتنقل.

## 5) Feature: Tafsir Workspace

الهدف:

- عرض تفسير الآية مع تنقل للآية السابقة/التالية.
- إبراز المصدر الحالي للتفسير.
- تمييز نصي ذكي (sacred names, terms, sections).

السلوك:

- تحميل التفسير حسب `verseKey` و`tafsirId`.
- عرض شارة المصدر عبر `tafsir-source-badge`.
- تنسيق النص إلى blocks (heading/paragraph) ثم تطبيق highlight.

نقاط تقنية:

- يدعم fallback عند تغير توفر مصدر التفسير.
- يقرأ verse sequence للتنقل المنطقي بين الآيات.

## 6) Feature: Settings + Persistence

الهدف:

- تخصيص التجربة (خط، وضع قراءة، قارئ، تفسير، theme، ...).
- حفظ كل الإعدادات محليًا بشكل دائم.

السلوك:

- أي تعديل يمر عبر `patchSettings`.
- القيم تستمر بعد refresh/reopen بفضل persisted stores.

## 7) Feature: Offline Mushaf Data

الهدف:

- تقليل الاعتماد على طلبات API الخارجية أثناء القراءة.
- تسريع استجابة endpoints الخاصة بالقراءة.

السلوك:

- يتم توليد `quran_offline.json` عبر سكربت download.
- endpoints مثل `/api/page` و`/api/rub` تعتمد على الملف المحلي.

## 8) تجربة المستخدم من البداية للنهاية

تدفق نموذجي:

1. المستخدم يضبط إعداداته في focus أو settings.
2. يبدأ جلسة دراسة.
3. ينتقل للقراءة (Rub أو Page).
4. يشغل التلاوة مع تتبع الكلمة النشطة.
5. يفتح التفسير للآية الحالية ويتنقل بين الآيات.
6. ينهي الجلسة وتُحتسب الإحصائيات.

## 9) معايير القبول (Acceptance Checklist)

- المؤقت ينتقل بين study/break بدون فقدان حالة.
- التنقل بين الأرباع/الصفحات لا يترك audio sessions عالقة.
- التلاوة تحدد active verse/word بشكل مرئي أثناء التشغيل.
- صفحة التفسير تظهر المصدر الحالي وتعرض النص بدون كسر تنسيق.
- الإعدادات تبقى محفوظة بعد إعادة التحميل.
